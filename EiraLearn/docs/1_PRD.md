# EiraLearn: Product Requirements Document (PRD)

## 📌 1. Problem Statement
Los candidatos al examen MEXT se estancan en rutinas de estudio pasivas (leer y repetir). El examen exige una extrema capacidad de asociación rápida, improvisación y dominio conceptual profundo (ej. mezclar física con geometría, o defender ideas abstractas en entrevistas). No existe una herramienta que entrene la *agilidad mental* específica para este nivel de estrés cognitivo.

## 👥 2. User Personas
- **Candidatos MEXT (Pregrado/Posgrado):** Estudiantes autodidactas con horarios apretados, bajo alto estrés, que necesitan optimizar su repetición espaciada y validar su conocimiento de forma agresiva. Buscan eficiencia absoluta sin gastar dinero.

## ⭐ 3. Core Features (MVP)
La app operará bajo 3 modos principales (Gimnasio Cognitivo):
1. **Modo Supervivencia (Síntesis):** El usuario recibe palabras aleatorias/del entorno y debe conectarlas creativamente para responder preguntas abiertas o simulaciones de entrevista. Entrena improvisación.
2. **Modo Colisión:** El usuario puede **pegar texto o subir imágenes** (ej. problemas matemáticos escaneados). La IA extrae 2 conceptos clave. El usuario debe fusionarlos en una regla, ecuación o solución creativa. Entrena cruce de disciplinas.
3. **Modo Socrático:** El usuario plantea una idea. La IA la cuestiona iterativamente (The 5 Whys), obligando al estudiante a re-explicar y re-estructurar su respuesta hasta llegar al concepto base irreductible.

*Nota:* Todas las interacciones permiten entrada multimodal (Pagar texto o Subir Imágenes de problemas).

## 📈 4. Success Metrics
- **Costo Cero Promedio:** La app no debe generar costos de infraestructura para el usuario (se debe limitar todo a Free Tiers).
- **Engagement:** Tiempo de permanencia por sesión (>15 minutos en debates socráticos).
- **Core Action:** Cantidad de "Colisiones" iniciadas por imágenes subidas.
